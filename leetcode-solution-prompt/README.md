# 🎓 Promlem Solving Prompt

## Why This Was Built

Every time I wanted to deeply understand a LeetCode problem —
not just the answer, but the *why* behind every single line —
I had to re-explain to AI how I wanted it taught.
Over and over. That was exhausting.

So I built this prompt once, saved it, and now I just paste a problem name.
Claude becomes my personal CS professor and teaches me everything I need.

---

## What It Does

You give it a LeetCode problem name or number.
It gives you a full university-level breakdown — instantly.

Every output includes:
- 📖 Plain English problem explanation with a real-life analogy
- 💡 3 approaches: Brute Force → Better → Optimal
- 🔬 Line by line teaching for every single solution
- 🔁 Manual iteration traces showing exactly how variables change
- 📊 Complexity comparison table across all 3 approaches
- ✅ Final clean commented optimal code
- 📌 A student rule note to keep you honest

---

## Who It's For

- 🐣 Beginners who want to understand the *why* behind every line
- 🧑‍🏫 Self-learners who want a professor, not just an answer
- 💼 Developers who want to deeply master DSA, not just memorize solutions

---

## How To Use

**Step 1** — Copy the full prompt from [`prompt.md`](./prompt.md)

**Step 2** — Paste it into Claude (claude.ai)

**Step 3** — Follow with your problem like this:
```
Problem: 49. Group Anagrams
```

**Step 4** — Get your full professor-style breakdown ✅

---

## What You'll Get (Example Structure)

```
1. Problem Understanding
   → Plain English + analogy + example

2. Brute Force
   → Intuition → Full Code → Line by Line → Iterations → Complexity

3. Better Approach
   → Intuition → Full Code → Line by Line → Iterations → Complexity

4. Optimal Approach
   → Intuition → Full Code → Line by Line → Iterations → Complexity

5. Comparison Table
   → All 3 side by side

6. Final Clean Code
   → Optimal solution, fully commented

7. Student Rule
   → No copy-pasting. Ever.
```

---

## The Student Rule

Every output ends with this — and it means it:

> 📌 **Student Rule:** Study each approach carefully and make sure you understand
> the intuition before coding. You are **prohibited from copy-pasting** any solution
> directly. Close the solution, open your editor, and write it yourself from scratch.
> Struggling is part of learning — embrace it. 💪
